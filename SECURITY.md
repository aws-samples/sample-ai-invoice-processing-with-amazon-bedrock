# Security Policy

## Reporting a Vulnerability

If you discover a potential security issue in this project, please **do not** create a public GitHub issue.

Instead, please report it by emailing the project maintainers. You should receive a response within 48 hours acknowledging receipt.

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |

## Security Best Practices

This project follows AWS security best practices including:
- IAM least privilege policies
- Encryption at rest and in transit
- Input validation and sanitization
- Bedrock Guardrails for AI safety
- Cognito authentication with MFA support
