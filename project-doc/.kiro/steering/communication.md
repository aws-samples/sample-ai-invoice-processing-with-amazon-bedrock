# Communication Guidelines

## Response Style

### Task Completion
- Provide brief confirmation when tasks are completed
- Avoid lengthy summaries or explanations unless specifically requested
- Use simple acknowledgments: "Done", "Created", "Updated", "Completed"

### Error Handling
- Report errors concisely with essential details only
- Focus on actionable information for resolution

### Code Changes
- Confirm file creation/modification without repeating the entire content
- Mention key changes only when relevant to next steps

### General Interaction
- Be direct and efficient in responses
- Save detailed explanations for when explicitly asked
- Prioritize action over commentary
